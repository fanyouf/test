题目1

为什么在安装了包之后，就可以像使用核心模块一样，直接通过require("包名")的方式来引用？

假设你在项目中安装了dayjs包，目前的目录结构是：

|-node_modules
|-node_modules/dayjs/...
|-index.js
|-utils/abc.js
|-package.json

问：在index.js和abc.js中是否也可以通过 const dayjs = require("dayjs") ，来使用dayjs包中的功能， 为什么？

答案

可以。原因是当require("dayjs")时，require的加载机制允许当前模块去搜索同级目录的node_modules下的dayjs，以及上级目录中node_modules下的dayjs，上上级目录中node_modules下的dayjs....。

你可以分别在index.js和abc.js下输出module.paths来验证这一点。

题目2

npm install 命令的作用是什么？

答案

安装package.json中dependencies及devDependencies属性中的包。

题目3

npm init --yes 的作用是?

答案

以默认的配置项生成package.json文件。

题目4

删除dayjs包的命令是？

答案

npm uninstall dayjs

题目5

如何理解npm?

答案

npm是随同nodejs一起安装的包管理工具。它同时也表示是一个网站，我们可以把自已代码以包的格式上传到这个网站，以供其它人下载使用。

题目6

如何理解npm中的包？

答案

英文名叫做 Packages，包是在模块基础上更深一步的抽象；用于更方便分发和推广代码； 包可以看作是 模块、代码 和 其它资源 组合起来形成的 独立作用域；包都要以一个单独的目录而存在，其中可以包含多个模块。package.json 必须在包的顶层目录下。
